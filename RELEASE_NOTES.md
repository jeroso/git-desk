## 🌿 v0.5.0 — 브랜치 패널 & 커밋 필터 개선

### ✨ 추가
- **커밋 필터** — 로그 상단 필터 바에서 **작성자**(자동완성) · **메시지 포함 텍스트** · **기간(시작일~종료일)** 으로 커밋을 걸러 볼 수 있습니다. IntelliJ 로그 필터처럼 여러 조건을 함께 적용합니다.
- **브랜치 다중 선택 & 일괄 삭제** — **⌘/Ctrl+클릭**(토글) 또는 **Shift+클릭**(범위)으로 여러 브랜치를 선택한 뒤, 우클릭 한 번으로 **한꺼번에 삭제**합니다. 로컬·원격 모두 지원하며, 병합되지 않은 브랜치는 모아서 강제 삭제 여부를 한 번 더 확인합니다.

### 🔧 변경
- **브랜치 전환은 우클릭 → Checkout 으로만** — 브랜치를 더블클릭하면 실수로 전환되던 동작을 없앴습니다. 클릭/더블클릭은 히스토리 보기만, 전환은 우클릭 메뉴에서만 합니다.
- **브랜치 폴더 기본 접힘 + 폴더 아이콘** — 폴더가 접힌 상태로 시작하고, **현재 체크아웃된 브랜치가 속한 폴더만 자동으로 펼쳐집니다.** 폴더 아이콘(📁/📂)과 굵은 글씨로 구조가 한눈에 들어옵니다.
- **체크아웃 충돌 시 스마트/강제 선택** — 수정 중인 파일이 있어 브랜치 전환이 막히면 IntelliJ처럼 **스마트 체크아웃**(변경을 대상 브랜치로 병합) 또는 **강제(하드) 체크아웃**(변경 버리고 전환)을 고를 수 있습니다. (한국어 git 환경에서도 정상 동작)

### 🐞 수정
- 커밋 필터로 커밋이 듬성해질 때 **그래프가 옆으로 무한정 넓어지고 커밋 메시지가 안 보이던** 문제를 고쳤습니다.
- 동시에 실행되는 git 명령이 서로의 인덱스 잠금(`index.lock`)과 경쟁해 **간헐적으로 실패하던** 문제를 방지했습니다(저장소별 순차 실행).
